# CLAUDE.md — Project Rules

## 1. Project
A quantitative **backtesting engine** for a multi-sleeve momentum + defensive trading strategy.
- **Mode:** research / paper-trade only — this code never places live orders.
- **Base currency:** CHF. **Venue:** PRIVATE. **Benchmark:** S&P 500 total return.
- **Goal:** measure the strategy's *real, overfitting-resistant* performance via leak-free backtesting and walk-forward out-of-sample validation.

## 2. Strategy summary
Multi-sleeve book (~14 sleeves), 3,500 CHF, intentionally decorrelated so combined Sharpe beats any single sleeve. Weights: MTUM 16%, RSP 11%, VLUE 11%, GLD 9%, dual-momentum rotation {RSP/EFA/GLD/BIL} 8%, USMV 7%, XLE 6%, XLP 5%, EUFN 5%, KO/PEP pair 4%, BTC 3%, SPY tail put 3%, VRP put spread 2%, cash/BIL 10%. Directional sleeves use a 2:1 reward:risk floor; the KO/PEP pair (z-score mean-reversion) and the VRP spread (50%-credit target) do not. Turnover ≤ 2 months. Daily VaR hard cap 15% (a cap, not a target; actual ~3–5%).

## 3. Architecture (where code goes)
Code lives in the `quant_backtest/` package, organized by pipeline stage. Create each subfolder only when building that phase:
- `config/` — universe, proxy map, paths, parameters
- `data/` — fetch (yfinance / Ken French / FRED), local store, FX conversion, quality checks
- `backtest/` — leak-free engine, CHF ledger, transaction costs
- `sleeves/` — signal logic, one module per sleeve
- `allocation/` — HRP / vol-target weighting, regime dial, leverage & VaR caps
- `walkforward/` — period split + out-of-sample harness
- `metrics/` — CHF risk/performance metrics
- `reporting/` — tables and plots

Tests live in `tests/` and mirror the package. Stored datasets live in the top-level `data/` folder and are **never committed** (regenerated by the ingestion script).

- Include explicit **no-look-ahead tests** for any signal or backtest code (see §7).

## 7. Quant guardrails — NON-NEGOTIABLE
These prevent the bugs that make a backtest lie. Treat them as hard rules.
- **No look-ahead:** signals use data only up to and including time *t*; trades execute on *t+1* (next bar). Never use the same bar's close for both signal and fill.
- **Trailing windows only:** z-scores, moving averages, volatility, betas use rolling/trailing data. Never normalize or fit using full-sample statistics.
- **Two-track history:** the strategy's instruments postdate 1998, so backtest on *proxies/factor series* for 1998–~2013 (logic test) and *real ETFs* ~2013→now (execution test). Annotate the splice; keep the out-of-sample holdout in the real-data track.
- **Currency:** all P&L and metrics in CHF — convert each USD price series via USDCHF before computing returns; convert the benchmark too. Sharpe uses a CHF risk-free rate.
- **Costs:** model Swissquote commissions and spreads. Never present gross-of-cost results as the headline.
- **Overfitting discipline:** minimize free parameters; walk-forward = fit on a training window, test on the next *unseen* window, roll; report only out-of-sample segments; check that results survive perturbing each threshold; limit how many variants are tried.
- **Reproducibility:** fixed seeds, pinned data-snapshot dates, data regenerated from the ingestion script.
